# CodeClarify AI – Product Requirements Document (PRD)

## 1. Product Overview

**Product Name:** CodeClarify AI
**Category:** AI Developer Tool / Learning Tool
**Primary Goal:** Help beginner programmers understand code quickly through structured AI explanations.

CodeClarify AI is a web application where users can paste code and receive clear, beginner-friendly explanations generated by AI. The system emphasizes **simplicity, structured explanations, and an easy interface for learners**.

Instead of requiring users to craft prompts manually (as they would in general AI chat tools), CodeClarify AI provides **one-click actions** such as explaining code, detecting bugs, optimizing code, and converting between programming languages.

The application focuses on **education and comprehension**, making it especially useful for beginner developers, students, and learners studying unfamiliar code.

The system will use a **separated full-stack architecture** consisting of a frontend web interface, a backend API server, and an AI inference provider.

---

# 2. Problem Statement

Beginner developers often struggle to understand unfamiliar code. While general AI chat systems can help, they present several usability challenges:

* Users must manually craft prompts.
* Responses may not follow a consistent explanation structure.
* Chat interfaces mix multiple responses, making code explanations harder to follow.
* General chat tools are not optimized specifically for code understanding workflows.

CodeClarify AI solves this by providing a **focused developer tool with structured explanations and one-click analysis actions**.

# 4. Core Product Features

## 4.1 Code Explanation

Users paste code and receive a structured explanation.

The response must always contain the following sections:

1. Simple Explanation
2. Line-by-Line Breakdown
3. Example Usage
4. Key Takeaways

The explanation must be written in **simple beginner-friendly language**.

---

## 4.2 Bug Detection

Users can analyze code to identify potential issues such as:

* logical bugs
* incorrect conditions
* edge case problems
* syntax mistakes

The AI will provide:

* bug description
* explanation of the problem
* suggested fix

---

## 4.3 Code Optimization

The AI analyzes the code and suggests improvements for:

* readability
* efficiency
* best practices

The response includes:

* improved version of the code
* explanation of improvements

---

## 4.4 Code Language Conversion

Users can convert code from one programming language to another.

Examples:

* Python → JavaScript
* Java → Python
* C++ → JavaScript

---

## 4.5 Beginner Mode (Important UX Feature)

The application includes a **Beginner Mode** that forces the AI to explain code in extremely simple terms.

Rules for Beginner Mode:

* avoid complex terminology
* use simple sentences
* explain programming concepts if needed
* provide small examples

This ensures the tool remains beginner-friendly.

# 6. Technical Architecture

The system uses a **decoupled frontend and backend architecture**.

Frontend handles UI and user interaction.
Backend handles API requests and AI communication.

---

## Architecture Diagram

User Browser
↓
Frontend (Next.js)
↓

## Frontend

Deployment: Vercel

---

## Backend

Runtime: Node.js
Framework: Express.js

Responsibilities:

* handle API requests
* validate user input
* communicate with Groq AI
* format responses

Deployment: Render

---

## AI Provider

Provider: Groq
Model: Llama 3

The backend will send prompts to Groq and return structured responses.

---

# 8. Backend API Design

Base URL:

/api

---

## Main Endpoint

POST /api/analyze

Purpose:

Analyze submitted code based on selected action.

---

### Request Body

{
"code": "user code here",
"action": "explain"
}

Supported actions:

explain
bugs
optimize
convert

---

### Response Format

{
"result": "AI generated explanation"
}

---

# 9. Input Constraints

To maintain performance and avoid exceeding AI model limits:

Maximum allowed code length:

350 lines of code

If the limit is exceeded, the API should return:

{
"error": "Code exceeds maximum allowed length."
}

---

# 10. Security Considerations

The Groq API key must never be exposed to the frontend.

Security measures:

* API key stored in backend environment variables
* Backend handles all AI requests
* Frontend never directly communicates with Groq
* Basic rate limiting on backend

---

# 11. Prompt Design

The backend will use a structured system prompt to ensure consistent responses.

Example instruction:

"You are an expert programming teacher. Explain the code in the simplest possible way so that a beginner can understand it."

The AI must structure responses with the following sections:

Simple Explanation
Line-by-Line Breakdown
Example Usage
Key Takeaways
